# Smart Study Hub 📘

**Smart Study Hub** — це сучасна система управління навчальним процесом (LMS), розроблена як курсовий проєкт. Головною особливістю є суворе дотримання принципів **Clean Architecture** (Uncle Bob) та реалізація **9 патернів проектування GoF**.

## 🚀 Технологічний стек
- **Backend:** [NestJS](https://nestjs.com/)
- **ORM:** [Prisma 7](https://www.prisma.io/)
- **Database:** PostgreSQL
- **Language:** TypeScript

---

## 🏛 Архітектура (Clean Architecture)
Проєкт розділений на незалежні шари, що забезпечує високу тестованість та гнучкість:

1.  **Domain Layer**: Чисті бізнес-сутності, інтерфейси репозиторіїв та логіка патернів. Жодних залежностей від фреймворків.
2.  **Application Layer**: Use Cases (сценарії використання), які оркеструють роботу домену.
3.  **Infrastructure Layer**: Реалізація репозиторіїв (Prisma), підключення до БД та зовнішніх сервісів.
4.  **Presentation Layer**: REST контролери, DTO та обробка HTTP-запитів.

## 🧩 Реалізовані патерни GoF (9/9)

### Creational (Породжувальні)
- **Factory Method**: `LessonFactory` — інкапсулює створення різних типів занять (Lecture, Lab, Seminar).
- **Builder**: `SubjectBuilder` — крок за кроком збирає складний об'єкт предмета з уроками та завданнями.
- **Singleton**: `DeadlineNotificationManager` — єдиний екземпляр для управління чергою сповіщень та WebSocket-з'єднаннями.

### Structural (Структурні)
- **Decorator**: `RecurringTaskDecorator` — додає функціонал автоматичного створення повторюваних завдань при завершенні поточних.
- **Facade**: `StudyHubFacade` — спрощує взаємодію з системою, надаючи єдиний метод для онбордингу користувача.
- **Composite**: `NoteComponent` — деревоподібна структура нотаток (розділи та блоки тексту).

### Behavioral (Поведінкові)
- **Command**: `ChangeTaskStatusCommand` — інкапсулює дію зміни статусу з можливістю скасування (Undo).
- **Observer**: `TaskStatusNotifier` — реактивне сповіщення системи про зміну стану завдань.
- **Strategy**: `ITaskSortStrategy` — можливість динамічної зміни алгоритму сортування списку завдань (за дедлайном або пріоритетом).

## 🛠 Налаштування та запуск

1. **Встановіть залежності:**
   ```bash
   npm install
   ```

2. **Налаштуйте базу даних:**
   Створіть файл `.env` та вкажіть `DATABASE_URL`.

3. **Згенеруйте Prisma Client:**
   ```bash
   npx prisma generate
   ```

4. **Запустіть додаток:**
   ```bash
   npm run start:dev
   ```

## 📡 API Ендпоінти

### 1. Онбординг нового студента (POST)
Створює стартовий предмет, лекцію та завдання для нового користувача за допомогою **Facade** та **Builder**.
```http
POST /onboarding
Content-Type: application/json

{ "userId": "student-id-123" }
```

### 2. Отримання предметів (GET - CQRS Query)
Повертає всі предмети користувача разом із завданнями та уроками.
```http
GET /subjects/user/student-id-123
```
